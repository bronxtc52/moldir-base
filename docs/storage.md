# Storage Model

Проект разделяет первоисточники, машинные сущности, индексы и будущие генерации.

## Слои

`knowledge/raw`

Сырье без ручной нормализации: исходный markdown продуктов, бренд-код, транскрибации, изображения. Этот слой нужен, чтобы всегда можно было вернуться к первоисточнику.

`knowledge/entities`

Структурированные карточки продуктов и наборов. Сейчас они генерируются из `knowledge/raw/products/products-full.md`, но позже часть полей можно будет редактировать вручную через админку или отдельный workflow.

`knowledge/indices`

Поисковый и сервисный индекс. Его читают HTTP API, MCP tools и будущие генераторы.

`knowledge/templates`

Шаблоны брифов и промптов. Здесь хранятся не готовые посты, а правила подготовки контента.

`knowledge/generated`

Выход контент-конвейера: статьи, посты, сценарии, видео-брифы, промежуточные черновики.

## Именование

- Для машинных файлов используются ASCII slug-и: `ashitaba-kapsuly.json`.
- Оригинальные имена файлов сохраняются в `source.path`.
- Изображения не переименованы на первом этапе, чтобы не потерять связь с исходной коллекцией.

## Правило источника правды

Если меняется продуктовая информация, сначала обновляется файл в `knowledge/raw`, затем запускается:

```bash
npm run index
```

Сгенерированные файлы не правятся вручную без отдельного решения.
